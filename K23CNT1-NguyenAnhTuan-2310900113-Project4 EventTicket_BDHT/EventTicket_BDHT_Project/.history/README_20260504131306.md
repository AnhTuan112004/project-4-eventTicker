# 🎟️ EVENT TICKET BOOKING PLATFORM (Hệ Thống Bán Vé Sự Kiện)

> Dự án xây dựng nền tảng thương mại điện tử chuyên biệt cho việc quản lý và đặt vé sự kiện trực tuyến. Dự án tích hợp 77 chức năng, áp dụng kiến trúc Client-Server với RESTful API và cơ chế chống quá tải vé (Optimistic Locking).

---

## 🛠️ 1. CÔNG NGHỆ SỬ DỤNG (Tech Stack)

### Backend
- **Ngôn ngữ:** Java 17
- **Framework:** Spring Boot 3.x (Spring Web, Spring Data JPA, Spring Security)
- **Bảo mật:** JWT (JSON Web Token), BCrypt Password Encoder
- **Database:** Microsoft SQL Server (12 bảng)

### Frontend
- **Ngôn ngữ:** HTML5, CSS3, Vanilla JavaScript
- **Giao tiếp API:** Fetch API 
- **Công cụ hỗ trợ UI:** Bootstrap / Tailwind CSS (Tùy chọn)

### Tools & Deployment
- **IDE:** Visual Studio Code (VS Code)
- **Quản lý mã nguồn:** Git & GitHub
- **Test API:** Postman

---

## 🌟 2. TÍNH NĂNG NỔI BẬT (Key Features)

Hệ thống được chia thành 2 phân hệ hoàn toàn độc lập với các chức năng cốt lõi:

**👤 Phân hệ Khách hàng (User/Member):**
- Đăng ký, Đăng nhập (Mã hóa mật khẩu).
- Tìm kiếm, lọc và xem chi tiết sự kiện.
- Đặt vé và thanh toán trực tuyến.
- Nhận vé điện tử (QR Code).
- Quản lý hồ sơ cá nhân (SD06) & Lịch sử đơn hàng.
- Đánh giá (Review) sự kiện sau khi tham gia.

**👑 Phân hệ Quản trị viên (Admin):**
- Quản lý người dùng (Khóa/Mở khóa tài khoản).
- Quản lý sự kiện, địa điểm, hạng vé.
- Quản lý mã giảm giá (Promotions).
- Thống kê doanh thu, số lượng vé bán ra theo thời gian thực.
- Kiểm duyệt vé tại cổng (Check-in).

🔥 **Đặc biệt (Technical Highlight):** Tích hợp thuật toán **Optimistic Locking** (Sử dụng cột `G8_version` trong bảng `ticket_types`) để xử lý triệt để bài toán đồng thời (Concurrency) - ngăn chặn lỗi bán lố vé (Overselling) khi có hàng ngàn user cùng mua 1 lúc.

## ⚙️ 3. HƯỚNG DẪN CÀI ĐẶT & CHẠY DỰ ÁN

### Yêu cầu môi trường (Prerequisites)
- [Java JDK 17+](https://www.oracle.com/java/technologies/javase/jdk17-archive-downloads.html)
- [Microsoft SQL Server](https://www.microsoft.com/en-us/sql-server/sql-server-downloads) & SSMS
- [Node.js & npm](https://nodejs.org/en/) (Để hỗ trợ Frontend nếu cần)
- VS Code với extension `Live Server`

### Bước 1: Cài đặt Database
1. Mở SQL Server Management Studio (SSMS).
2. Tạo database mới với tên: `Event_Ticket_BDHT`.
3. Chạy script SQL (đính kèm trong thư mục `database/`) để tạo 12 bảng dữ liệu và chèn dữ liệu mẫu (Mock data).

### Bước 2: Chạy Backend (Spring Boot)
1. Mở thư mục `backend-event-ticket` bằng IDE.
2. Mở file `src/main/resources/application.properties`.
3. Cập nhật `username` và `password` của SQL Server máy bạn:
   ```properties
   spring.datasource.url=jdbc:sqlserver://localhost:1433;databaseName=Event_Ticket_BDHT;encrypt=true;trustServerCertificate=true
   spring.datasource.username=SA_CUA_BAN
   spring.datasource.password=PASS_CUA_BAN
