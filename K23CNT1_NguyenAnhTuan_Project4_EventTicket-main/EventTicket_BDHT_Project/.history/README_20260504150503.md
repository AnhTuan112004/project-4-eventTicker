## 2. PHÂN CÔNG ĐỘI NGŨ (Team Structure)

### 👨‍💼 Đội Backend (2 người)

#### **Thành Viên 1: Backend Lead - API & Database**
- **Trách Nhiệm**:
  - Thiết kế và phát triển RESTful API (Spring Boot)
  - Quản lý kết nối Database (MS SQL Server)
  - Xây dựng các Entity JPA (User, Event, Booking, Ticket)
  - Implement Optimistic Locking cho tính năng booking vé
  - Cấu hình Database: Migration, Indexes, Constraints
  - Xử lý lỗi và exception handling

- **Công việc cụ thể**:
  - `backend/src/main/java/com/eventticket/entity/` - Các Entity class
  - `backend/src/main/java/com/eventticket/repository/` - JPA Repositories
  - `backend/src/main/java/com/eventticket/controller/` - API Controllers
  - `database/event_ticket.sql` - Database schema

#### **Thành Viên 2: Backend Developer - Authentication & Business Logic**
- **Trách Nhiệm**:
  - Implement JWT Authentication (Login/Register)
  - Phát triển Service Layer (Business Logic)
  - Xây dựng chức năng booking, thanh toán, review
  - Cấu hình Security (Spring Security + JWT)
  - Tối ưu hóa queries và performance
  - Unit Testing cho Backend

- **Công việc cụ thể**:
  - `backend/src/main/java/com/eventticket/service/` - Business Logic
  - `backend/src/main/java/com/eventticket/security/` - JWT Util, Security Config
  - `backend/src/main/java/com/eventticket/exception/` - Exception Handlers
  - `backend/src/test/` - Unit Tests

### 👨‍🎨 Đội Frontend (2 người)

#### **Thành Viên 3: Frontend Lead - User Interface**
- **Trách Nhiệm**:
  - Thiết kế giao diện người dùng (User Pages)
  - Phát triển trang home, danh sách sự kiện, chi tiết sự kiện
  - Xây dựng trang booking vé và thanh toán
  - Tạo trang profile người dùng
  - Responsive design (Mobile, Tablet, Desktop)
  - CSS/Styling với Bootstrap hoặc Tailwind

- **Công việc cụ thể**:
  - `frontend/pages/` - HTML files (index.html, events.html, booking.html, profile.html)
  - `frontend/css/` - Styling sheets
  - `frontend/assets/` - Images, icons, media
  - `frontend/docs/` - UI/UX documentation

#### **Thành Viên 4: Frontend Developer - Admin Dashboard & API Integration**
- **Trách Nhiệm**:
  - Phát triển Admin Dashboard
  - Tích hợp Fetch API với Backend
  - Xây dựng JavaScript logic (Form handling, Validation)
  - Quản lý JWT tokens (LocalStorage/SessionStorage)
  - Implement QR Code generation & scanning
  - Performance optimization & Caching

- **Công việc cụ thể**:
  - `frontend/pages/admin/` - Admin Dashboard pages
  - `frontend/js/` - JavaScript logic (api.js, auth.js, qrcode.js, utils.js)
  - `frontend/js/components/` - Reusable components
  - `frontend/tests/` - Frontend testing

### 🔐 JWT Authentication Flow

```
1. User đăng nhập
   POST /api/auth/login (email, password)
                ↓
2. Backend xác thực thông tin
   - Kiểm tra user trong database
   - Verify password (BCrypt)
                ↓
3. Tạo JWT Token
   - Header: {"alg": "HS256", "typ": "JWT"}
   - Payload: {"userId": 1, "email": "user@email.com", "role": "USER", "exp": ...}
   - Signature: HMAC-SHA256(header.payload, secret)
                ↓
4. Return token to client
   Response: {"token": "eyJhbGc...", "user": {...}}
                ↓
5. Client lưu token vào LocalStorage
                ↓
6. Requests tiếp theo gửi token
   Header: Authorization: Bearer eyJhbGc...
                ↓
7. Backend verify token
   - Validate signature
   - Check expiration
   - Extract user info (userId, role)
                ↓
8. Cho phép hoặc từ chối request
   - Nếu token hợp lệ → Tiếp tục
   - Nếu token hết hạn → Return 401 Unauthorized
```

2. **Backend**: `cd backend && mvn spring-boot:run`.
3. **Frontend**: Open `frontend/index.html` with Live Server.
4. **Test**: Postman -> `http://localhost:8080/api/events`.
